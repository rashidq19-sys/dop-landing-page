# DSPOps Landing Page Redesign — Design Spec

**Date:** 2026-04-02
**Status:** Approved
**Approach:** Problem → Solution flow, audience-segmented features

---

## Overview

Restructure the DSPOps marketing landing page to better communicate value to the primary audience (DSP owners and OSMs). The current page lists features individually; the new design groups them by audience (Owner, OSM, Driver portal) and adds new sections: a morning dispatch storyboard, before/after comparison, and an interactive cost calculator. Fake testimonials are removed. CTAs shift from "Start Free Trial" to "Get Early Access" + "Book a Demo" since sign-up and Calendly are not yet set up.

---

## Target Audience

- **Primary:** DSP owners — care about visibility, control, cost savings, scaling
- **Secondary:** OSMs — care about morning dispatch efficiency, no spreadsheets, time savings
- **Not visiting the page:** Drivers — but the "driver portal" section is a selling point TO owners/OSMs ("your drivers stop coming to you")

## Technical Notes

- **Cost Calculator:** Client-side only. No API. Simple multiplication based on inputs. Use React state for interactivity.
- **Email capture:** The "Get Early Access" forms need a backend endpoint or third-party integration (e.g., Mailchimp, ConvertKit) to store emails. This is out of scope for the landing page redesign but needs to be connected.
- **Demo form:** Simple form submission (name, email, DSP size). Can store in same system as email capture or send as email notification.
- **Existing video:** Reuse current MP4 from CloudFront. No new video needed.
- **Existing screenshots:** Reuse current CloudFront-hosted images. May need additional screenshots for Driver Portal section if available.
